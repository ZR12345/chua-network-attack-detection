# Resilient Attack Detection and Priority-Based Protection in Synchronized Networks

A simulation framework for studying cyber-physical attacks on synchronized
multi-agent networks, built around a network of coupled **Chua's circuit**
nodes. The project extends a base synchronization model with two original
contributions: a **real-time attack detection module** and a **priority-based
node protection scheme**, evaluated using standard detection-performance
metrics (ROC curve, AUC, confusion matrix).

## Overview

Distributed cyber-physical systems — sensor networks, power grids, multi-robot
systems — rely on nodes staying synchronized with each other and with a shared
target trajectory. When an attacker disrupts the communication topology
(severing or altering links between nodes), synchronization degrades and the
system can become unreliable or unsafe.

This project simulates an 8-node network where each node follows **Chua's
circuit** dynamics (a well-known chaotic nonlinear system) and is coupled to
its neighbors through a shared network topology. On top of a baseline
synchronization/observer design, this implementation adds:

- **Novelty 1 — Attack detection:** a residual-based statistical detector
  that monitors temporal and spatial inconsistencies between nodes and flags
  attacks in real time, without prior knowledge of when or where an attack
  will occur.
- **Novelty 2 — Priority-based protection:** node importance is computed from
  network centrality (betweenness, degree, or PageRank), and control effort is
  reallocated so that high-priority nodes synchronize faster and recover more
  quickly when the network is under attack.

## How it works

1. **Network topology** — an 8-node graph defines normal connectivity between
   nodes, with a corresponding graph Laplacian.
2. **Attack topologies** — four attack scenarios (a–d) simulate progressively
   more severe link failures, from a single broken connection to widespread
   damage.
3. **Chua's circuit dynamics** — each node's local (uncoupled) behavior
   follows Chua's circuit equations, with a target trajectory generated by
   solving the isolated system.
4. **Synchronized network simulation** — all 8 nodes are simulated together
   under a randomly generated attack sequence that switches between normal and
   attack topologies over time. Each node also runs a local observer that
   estimates its own state from limited measurements.
5. **Attack detection (Novelty 1)** — at each time step, a detector computes a
   residual signal from temporal state changes and spatial disagreement
   between neighboring nodes, and compares it against a threshold to decide
   whether an attack is in progress.
6. **Priority-based protection (Novelty 2)** — node priority weights (derived
   from graph centrality) scale each node's control gain, so structurally
   important nodes receive more control effort and stabilize faster.
7. **Performance evaluation** — detection performance is assessed with a ROC
   curve and AUC score, and a confusion matrix reports accuracy, precision,
   recall, and F1-score against ground-truth attack intervals.

## Getting started

### Prerequisites

- Python 3.9+
- Jupyter Notebook, JupyterLab, or VS Code with the Jupyter extension

### Installation

Clone the repository and install dependencies:

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
cd YOUR_REPO_NAME
pip install -r requirements.txt
```

### Running the simulation

Open and run the notebook top to bottom:

```bash
jupyter notebook CyberSec_2.ipynb
```

Running all cells in order will:
1. Build the network and attack topologies
2. Generate the target trajectory
3. Run the full synchronized simulation under a randomized attack sequence
4. Compute detection and priority-protection results
5. Produce the ROC curve and confusion matrix
6. Save all result figures into the `results/` folder (final cell)

## Key results

The notebook reports the following at the end of its run (exact values may
vary slightly between runs due to the randomized attack sequence and noise
in initial conditions):

| Metric | Description |
|---|---|
| Detection rate | Percentage of true attacks correctly flagged |
| False alarm rate | Percentage of false positives during normal operation |
| Average detection delay | Time between attack onset and detection |
| ROC AUC | Area under the ROC curve for the detector |
| Accuracy / Precision / Recall / F1 | Standard classification metrics from the confusion matrix |
| Average convergence time | Time for nodes to synchronize with the target trajectory |

See `results/06_roc_curve.png` and `results/07_confusion_matrix.png` for the
specific numbers from the latest run, and `results/05_sync_performance.png`
for how high-priority nodes recover faster than low-priority ones during
attacks.

## Methodology notes

- **Base model:** an 8-node network of Chua's circuits, coupled through a
  shared topology, with a local Luenberger-style observer at each node
  estimating its state from partial measurements.
- **Attack model:** topology-switching attacks — an adversary severs or
  rewires links between nodes for randomly generated time intervals, as
  defined in the attack-sequence generator.
- **Detector design:** a residual signal combining the temporal rate of
  change in each node's estimated state and its spatial disagreement with
  neighboring nodes, thresholded at an empirically tuned value.
- **Priority weighting:** centrality-based weights (normalized to a [0.3, 1.0]
  range to avoid fully de-prioritizing any node) scale each node's control
  gain during normal operation.

## Limitations

- Attack scenarios and detection thresholds are tuned for this specific
  8-node topology and parameter set; results will differ for other network
  sizes or dynamics.
- The observer gain matrices used here are precomputed (not derived live via
  LMI solvers) for demonstration purposes.
- This is a simulation study, not a deployed or hardware-validated system.
